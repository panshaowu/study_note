# 测试时发现学习 (Learning to Discover at Test Time)

**作者**: Mert Yuksekgonul, Daniel Koceja, Xinhao Li, Federico Bianchi, Jed McCaleb, Xiaolong Wang, Jan Kautz, Yejin Choi, James Zou, Carlos Guestrin, Yu Sun
**开源仓库**: [https://github.com/test-time-training/discover](https://github.com/test-time-training/discover)

## 摘要 (Abstract)

我们如何利用AI为科学问题发现新的最先进（state of the art）解决方案？在此之前的测试时扩展（test-time scaling）工作，例如 AlphaEvolve，通过提示一个冻结的大语言模型（LLM）来进行搜索。我们在测试时执行强化学习，使得 LLM 可以继续训练，但此时使用的是特定于测试问题的经验。这种形式的持续学习非常特殊，因为它的目标是产生一个极佳的解决方案，而不是在平均水平上产生多个好的解决方案，并且是为了解决这个特定问题，而不是泛化到其他问题。因此，我们的学习目标和搜索子程序被设计为优先考虑最有前途的解决方案。我们将此方法称为“测试时发现训练”（Test-Time Training to Discover，TTT-Discover）。继先前的工作之后，我们专注于具有连续奖励的问题。

我们报告了我们尝试过的每一个问题的结果，涵盖了数学、GPU内核工程、算法设计和生物学。TTT-Discover 在几乎所有这些领域都确立了新的最先进水平：（i）Erdős 的最小重叠问题和自相关不等式；（ii）GPUMode 内核竞赛（比此前的算法快最高2倍）；（iii）过去的 AtCoder 算法竞赛；（iv）单细胞分析中的去噪问题。我们的解决方案已由专家或组织者审核。

我们所有的结果都是使用开源模型 OpenAI gpt-oss-120b 取得的，并且可以使用我们公开的代码进行重现，这与以前需要闭源前沿模型才能取得最佳结果的工作形成了鲜明对比。我们的测试时训练运行是使用 Thinking Machines 提供的 Tinker API 完成的，每个问题的成本仅为几百美元。

## 1. 引言 (Introduction)

为了解决困难的问题，人类通常需要尝试、失败、偶然取得部分成功，然后从经验中学习。想象一下你第一次遇到真正困难的编程作业。你阅读了教科书并在书本练习上进行了自我训练，但这个作业的要求远远超出了书本中的基础知识。你尝试猜测解决方案，但这些尝试仅仅产生了一点微小的生机。所以你必须深吸一口气，从失败的尝试中学习，这使你未来的尝试变得更加明智。最终，在经过数小时的尝试和学习后，你理解了作业背后的新思路。果然，下一次尝试成功了！

在这个例子中，作业之所以困难，是因为它需要超出你的训练数据（教科书和书本中的练习）的新思路。现在考虑使用 AI 来解决科学发现问题。这个目标甚至更加困难：顾名思义，发现问题不仅需要超出模型训练数据的思路，还需要超出人类所有现有知识的思路。而且，分布外泛化（out-of-distribution generalization）对 AI 来说并不比对人类容易。

为了抵消这种困难，先前的工作主要集中在解空间中的测试时搜索上，通过提示一个冻结的 LLM 进行多次尝试，类似于我们试图猜测作业解决方案的方式。特别是进化搜索方法，如 AlphaEvolve，将过去的尝试存储在缓冲区中，并通过手工制作和特定领域的启发式方法使用它们来生成新的提示。虽然这些提示可以帮助 LLM 改进以前的解决方案，但 LLM 本身无法改进，就像一个永远无法将作业背后的新思想内化吸收的学生一样。

LLM 改进的最直接方式是通过学习。确实，尽管学习和搜索都可以随着计算量的增加而很好地扩展，但在 AI 的历史中，对于围棋和蛋白质折叠等难题，学习往往取代了搜索。我们相信，随着我们在测试时扩展计算，这段历史的观察在今天仍然具有现实意义。因此，在 LLM 尝试解决这个测试问题时，我们继续对其进行训练。而这些尝试反过来又提供了最有价值的训练数据：回想一下，测试问题之所以难，是因为它是分布外的。现在我们有了特定于这个问题的数据分布。

在高层次上，我们只是在一个由单个测试问题定义的环境中执行强化学习 (RL)，因此标准 RL 中的任何技术都可以应用。然而，我们的目标与标准 RL 有两个关键的区别。首先，我们的策略只需要解决这一个问题，而不是泛化到其他问题。其次，我们只需要一个最佳解决方案，而策略只是实现这一目标的手段。相反，在标准 RL 中，策略是最终目的，其目标是最大化所有尝试的平均奖励。虽然第一个区别是测试时训练领域的常见主题，但第二个区别是发现问题所独有的。

为了利用这些差异，我们的学习目标和搜索子程序强烈偏向于最有前途的解决方案。我们将此方法称为“测试时发现训练” (TTT-Discover)。我们专注于数学、GPU内核工程、算法设计和生物学中具有连续奖励的问题。我们报告了我们尝试过的每一个问题的结果，TTT-Discover 仅使用一个开源模型就在几乎所有问题中设定了新的最先进水平。

有三项同时进行的工作与我们的高层思想相同：EvoTune、MiGrATe，以及最近的 ThetaEvolve，后者尤为相关。与 ThetaEvolve 相比，TTT-Discover 使用相同的模型和计算预算仍然产生了显著的改进，这归功于其特殊的学习目标和搜索子程序。



## 2. 预备知识 (Preliminaries)

本文中的所有方法，包括基线方法，都有一个共同的目标：在测试时给定一个科学问题，目标是使用一个 LLM 策略 $\pi_\theta$ 发现一个新的最先进解决方案，该策略的权重 $\theta$ 已经在训练时被训练过。为了形式化这个目标，我们首先介绍每个科学问题如何定义一个环境，即马尔可夫决策过程（Markov Decision Process, §2.1），然后该过程可以用于搜索（§2.2）和学习（§3）。

### 2.1 发现问题 (Discovery Problem)

我们对环境的定义遵循先前在测试时扩展（test-time scaling）方面的工作，例如 AlphaEvolve：科学问题以文本描述 $d$ 的形式出现，我们始终将其作为上下文提供给策略。我们将状态 $s$ 定义为候选解决方案，例如 $d$ 中 PyTorch 代码的内核实现。在我们的应用中，问题描述还诱导了一个连续的奖励函数 $R(s) \in \mathbb{R}$，例如内核反向运行时间（inverse runtime）。

我们将 $s_{sota}$ 记为所有现有候选方案中已知最好的解决方案，并将 $r_{sota} = R(s_{sota})$ 记为已知最好的奖励。如果在没有现有解决方案的情况下，$s_{sota}$ 可以是空字符串 `<empty>`。例如，$s_{sota}$ 可以是当前排行榜顶部的内核。这些符号使我们能够形式化发现的概念：

**定义（发现）**。发现是一个事件，其中找到了一个状态 $s$，使得 $R(s) > r_{sota}$。差异越大，发现就越重要。

在这种形式化下，我们将发现问题定义为在科学问题定义的环境中找到具有较大 $R(s) - r_{sota}$ 的这样一个状态 $s$。

为了产生更好的解决方案，搜索和学习方法都使用 LLM 策略来生成动作 $a \sim \pi_\theta(\cdot | d, s)$，其中初始解决方案 $s$（例如，$s = s_{sota}$）的选择是方法设计的一个重要部分。与奖励函数类似，环境的转移函数 $(s, a) \rightarrow s'$ 也是由问题描述诱导的。在这里，我们只考虑一个单步时间步长，因为状态重用（state reuse，我们将很快介绍）有效地包含了多个时间步长。

在我们的所有应用中，一个有效的动作包含一段代码，并且可选地包含一些思考（thinking）标记。对于编码问题（例如内核工程），环境通过简单地从 $a$ 中解析出代码来产生 $s'$。对于数学问题，环境还需要在解析出 $a$ 中的代码后执行它。表1概述了我们所有应用的环境。

### 2.2 搜索方法 (Search Methods)

最简单的搜索方法称为 Best-of-N，从 $\pi_\theta$ 采样独立同分布的 rollout（展开）：
**Best-of-N**: $s = s_{sota}$ 或 `<empty>`, $a_i \sim \pi_\theta(\cdot | d, s)$,
其中下标 $i = 1, \dots, N$ 表示 rollout 的索引。通过使用 $i$ 而不是 $t$ 作为索引，我们表示此处的 rollout 是独立的。假设存在先前的解决方案，初始状态 $s$ 的一个合理选择是 $s_{sota}$。但 $s_{sota}$ 可能是一个过于强烈的利用（exploitation）先验。例如，以 $s_{sota}$ 为条件可能会阻止策略探索非常不同但更有前途的方向，这些方向最终将在大的计算预算下产生更好的解决方案。为了解决这种担忧，我们通常设置 $s = \text{<empty>}$，即空（或平凡的）解决方案。

另一方面，策略也可能使用 $s = \text{<empty>}$ 探索一个有前途的方向，但未能充分利用它。解决这种相反担忧的一种技术是状态重用（state reuse），它使用一些以前的解决方案来热启动（warm start）策略。具体来说，它维护一个以前解决方案的缓冲区 $\mathcal{H}_i$，并使用一个搜索启发式方法 `reuse` 从 $\mathcal{H}_i$ 中采样初始解决方案 $s_i$，该启发式方法偏好高奖励解决方案，但仍然将非平凡的可能性分配给低奖励的解决方案：
**State reuse**: $s_i \sim \text{reuse}(\mathcal{H}_i)$, $a_i \sim \pi_\theta(\cdot | d, s_i)$, $\mathcal{H}_{i+1} = \mathcal{H}_i \cup \{(s'_i, r_i)\}$.
当我们重用以前的解决方案 $s'_i$ 时，我们实际上在其轨迹上添加了一个额外的时间步。

先前的工作（如 AlphaEvolve）也重用了可能包含不属于状态一部分的思考标记和中间结果（例如数学问题的代码）的动作。因此，`reuse` 启发式方法还需要将先前动作的信息转换为可由 LLM 策略接收的自然语言上下文 $c_i$：
**State-action reuse**: $s_i, c_i \sim \text{reuse}(\mathcal{H}_i)$, $a_i \sim \pi_\theta(\cdot | d, s_i, c_i)$, $\mathcal{H}_{i+1} = \mathcal{H}_i \cup \{(s_i, a_i, s'_i, r_i)\}$.
以前的工作将状态-动作重用称为进化搜索，因为重用启发式方法通常涉及受进化启发的复杂设计，包括手工制作的变异（mutation）和交叉（cross-over）操作，以及特定领域的适应度（fitness）和多样性（diversity）测量。

## 3. 测试时发现学习 (Learning to Discover at Test Time)

到目前为止，策略在测试问题上的经验只能改进下一个提示 $(d, s_i, c_i)$，而不能改进策略 $\pi_\theta$ 本身，因为 $\theta$ 保持冻结。我们利用这种经验，通过在累积在缓冲区 $\mathcal{H}_i$ 中的自身搜索尝试上训练 $\pi_\theta$，以在线（online）方式改进策略。

### 3.1 测试时的朴素强化学习 (Naive RL at Test Time)

算法1属于强化学习（RL）的公式。一个自然的基线是使用标准的 RL 算法：
**train**: $\theta_{i+1} = \theta_i + \eta \nabla_\theta \mathbb{E}_{a \sim \pi_{\theta_i}(\cdot|s)}[R(s, a)]$,  **reuse**$(\mathcal{H}_i) = \delta_{\text{<empty>}}$,
即优化不使用重用的期望奖励，其中 $\delta_{\text{<empty>}}$ 是仅在初始状态 `<empty>` 上具有质量的 delta 分布。我们将使用 $\theta_i$ 来表示 rollout $i$ 的模型权重。我们可以直接应用流行的 RL 算法，如 PPO 或 GRPO，仅在由单个问题定义的环境中进行。

然而，这些算法是在考虑标准 RL 问题的情况下设计的。发现问题与标准 RL 问题有重要的区别：
* **标准 RL 问题**：目标是找到最大化期望奖励的策略。该策略将部署在同一个环境中反复执行。**主要的产物是策略（Policy）**。
* **发现问题 (Discovery Problem)**：目标是找到一个改进最先进技术（SOTA）的单一状态（State）。**不在乎平均表现**。这里没有单独的部署阶段，因此策略不需要在从相同初始状态分布开始时可能遇到的许多状态中保持稳健的性能。事实上，一个策略的期望奖励可能非常低，只要它达到一次新的 SOTA 即可突破。

由于上述差异，朴素的 RL 应用于发现问题时具有致命的缺点：
* **目标函数 (Objective function) 错位**：朴素的 RL 优化平均表现，对 SOTA 漠不关心。然而在发现任务中，成功是由**最大值**决定的。考虑一个内核工程问题，SOTA 运行时间是 2000$\mu s$，达到 1900$\mu s$ 可能是一项突破。但如果没有复杂的奖励塑造，两者在传统的平均视角下将获得几乎相同的奖励。
* **有效视界短 (Short effective horizon)**：每次尝试都从头开始，极大限制了策略在单次展开中所能达到的深度。**重用 (Reuse)** 先前的解决方案实际上为一次尝试添加了额外的时间步长，从而扩展了视界。在标准 RL 中，固定的初始状态分布是为了稳健性，而发现任务没有部署阶段，无需这种约束。
* **探索 (Exploration) 保守**：优化期望奖励时，策略可能会“趋利避害”，坍缩为安全的、高回报的动作，而不是去尝试可能打破 SOTA 的高风险动作。在重用层面上，朴素的优先级排序可能会过度利用少数有希望的状态而牺牲了多样性。

### 3.2 TTT-Discover

为了解决传统 RL 在发现问题上面临的缺点，TTT-Discover 引入了两个简单但至关重要的组件：**熵目标（优化最大值）**和**基于最大值的 PUCT（深化探索）**。

#### 1. 熵目标 (Entropic Objective)：强行扭转优化方向，只盯最高分
传统的 RL 旨在优化“期望（平均）奖励”，但这无法引导模型去寻找单一的突破性 SOTA。为此，我们定义了偏好最大奖励动作的熵目标：
$$ J_\beta(\theta) = \mathbb{E}_{s \sim \text{reuse}(\mathcal{H})} \left[ \log \mathbb{E}_{a \sim \pi_\theta(\cdot|s)} \left[ e^{\beta(s)R(s, a)} \right] \right] $$
其梯度为：
$$ \nabla_\theta J_\beta(\theta) = \mathbb{E}_{s \sim \text{reuse}(\mathcal{H})} \left[ \mathbb{E}_{a \sim \pi_\theta(\cdot|s)} \left[ w_{\beta(s)}(a) \nabla_\theta \log \pi_\theta(a|s) \right] \right] $$
其中更新权重 $w_{\beta(s)}(a) = \frac{e^{\beta(s)R(s,a)}}{\mathbb{E}_{\pi_\theta(\cdot|s)}[e^{\beta(s)R(s,a)}]}$。此外，我们还使用 KL 惩罚项来塑造优势函数，并发工作也探索了类似目标以最大化 pass@k 性能。

**为什么用指数函数？（“赢者通吃”效应）**
将奖励 $R$ 放到指数 $e$ 的肩膀上（$e^{\beta \cdot R}$），会产生强大的数学放大效应。假设模型做了三次尝试，得分分别是 1分(平庸)、2分(不错)、10分(破纪录)。在传统 RL 的线性视角下，它们的贡献是 (1, 2, 10)。但放入指数后（设 $\beta=1$），它们变成了 $e^1 \approx 2.7$、$e^2 \approx 7.4$、$e^{10} \approx 22026$。
破纪录的 10 分瞬间被放大到 **22026**，完全碾压了平庸的分数。反映在梯度更新权重 $w_{\beta(s)}(a)$ 上，模型会**几乎完全忽略平庸的尝试**，把绝大部分的学习精力砸向那个得 10 分的动作。

在数学上，当 $\beta \rightarrow \infty$ 时，这个熵目标严格等价于求最大值 $\max(R)$，这正是发现任务所追求的。然而，训练早期过大的 $\beta$ 会导致不稳定性，而后期过小则会使优势消失。因此，我们通过约束 KL 散度，为每个初始状态自适应地设置 $\beta(s)$（详见附录 A.1）。

#### 2. PUCT 状态重用：站在“巨人的肩膀上”继续探索
为了解决每次从头开始导致“视野短”的问题，我们使用历史记录缓冲区 $\mathcal{H}$ 来保存过去的尝试，并使用受 PUCT 启发的规则从中选择一个优秀的“半成品”状态 $s$ 作为新的起点。每个状态 $s$ 的得分为：
$$ \text{Score}(s) = Q(s) + c \cdot P(s) \cdot \frac{\sqrt{1 + T}}{1 + n(s)} $$
其中 $P(s)$ 是基于排名的先验概率，$n(s)$ 是状态被展开的次数，$T$ 是总展开数，$c$ 是探索系数。

**关键修改：用子代最大奖励替代平均奖励**
> 💡 **注：什么是标准 PUCT？**
> PUCT (Predictor + Upper Confidence Bound applied to Trees) 是 AlphaGo 等著名 AI 系统中广泛使用的启发式树搜索算法。它的核心思想是平衡 **“利用 (Exploitation)”**（选择历史得分高的节点）和 **“探索 (Exploration)”**（选择访问次数少、但先验潜力大的节点）。在标准 PUCT 中，评估一个节点历史得分的指标 $Q(s)$，通常是基于该节点衍生出的所有探索结果的**平均奖励**。

与标准 PUCT 使用子节点的**平均奖励**不同，此处的 $Q(s)$ 定义为从状态 $s$ 生成的子状态中的**最大奖励**。因为在发现问题中，我们关心的是从一个状态开始所能达到的**最佳上限**，而不是平均表现。
这种先验设计捕捉到了一种直觉：一个高奖励的半成品代码（比如一个已经很快的内核），比一个低奖励的代码更有可能孕育出破纪录的后代。PUCT 引导模型在“曾经孕育出极高分”的路径上持续深挖，而不浪费算力在平庸的方向上。

> 💡 **核心机制补充：上下文如何避免被撑爆？**
> 对于任何一次单独的探索（rollout），LLM 看到的上下文（Prompt）中**绝不会**包含整个历史记录。系统仅仅将**唯一一个**被 PUCT 选中的“半成品”（且只提取纯代码，**彻底丢弃上一轮长篇大论的“思考过程”**）作为初始状态放入 Prompt。这种“每次只放入一段精简代码”的设计，使得平均提示词长度仅为 3000 Tokens，为 LLM 留下了极其充裕的上下文空间去进行新的深度思考，完美避开了早期进化搜索算法（如早期的 AlphaEvolve）极易导致上下文爆炸的致命缺陷。

#### 3. 测试时发现训练 (Test-time Training to Discover)
有了这两个组件，我们就可以定义 TTT-Discover 的完整工作流。我们将熵目标 $J_{\beta(s)}$ 结合为测试时的在线训练目标，将 PUCT 作为状态重用子程序：
* **train**: $\theta_{i+1} = \theta_i + \eta \nabla_\theta J_{\beta(s_i)}(\theta_i)$
* **reuse**: $s_i \sim \text{PUCT}(\mathcal{H}_i)$

**总结**：模型实时从历史中挑选最有希望的半成品 (PUCT)，顺着它继续尝试 (Rollout)，然后用“赢者通吃”的目标函数计算梯度，直接现场更新自身权重 (Train)，不断逼近 SOTA。

#### 4. 算法局限性：Problem-Specific (不可跨任务迁移)
由于 TTT-Discover 本质上是在**单个测试环境（即单个特定任务）**中进行强化学习，其训练出的模型权重是**高度 Problem-Specific（问题特异性）的**。
- **目标仅为单一最优解**：模型微调的目的是为了寻找当前给定问题（如特定的 GPU Kernel 优化、特定的数学猜想）的单一 SOTA 解决方案，它不需要也不具备泛化能力。
- **权重无法跨任务复用**：如果需要解决一个全新的任务（例如从优化 TriMul 算子变为优化 MLA 算子），必须**丢弃**在先前任务上过拟合的模型权重，重新使用干净的初始预训练大模型（如 `gpt-oss-120b`），并在新任务的测试环境中从零开始重新执行 TTT-Discover 的微调过程。这意味着每探索一个新的发现任务，都需要独立支付一次完整的测试时搜索和训练的计算成本。

### 3.3 实现细节 (Implementation Details)

为了让上述复杂的测试时训练能够高效且经济地运行，论文在实现上做了一系列精心设计的工程取舍。以下是结合前文原理的具体执行细节：

#### 1. 训练环境与成本
- **模型与平台**：全程使用开源大模型 `gpt-oss-120b`，并在 Thinking Machines 提供的 Tinker API 平台上运行。
- **训练成本极低**：假设平均提示词长度为 3000 个 Tokens，平均每次生成 16000 个 Tokens，**一次完整的测试时训练（包含 50 个训练步，每步 512 次模型采样）总成本仅约 500 美元**。

#### 2. TTT-Discover 的单步执行循环
整个 TTT-Discover 算法会循环执行 **50 个训练步骤 (steps)**。在每一个步骤中，具体的细节如下：
- **高效微调 (LoRA)**：为了避免在测试阶段全量更新千亿参数模型的巨大开销，模型采用了秩 (rank) 为 32 的 LoRA（低秩适配）技术来进行权重的现场更新。
- **结合 PUCT 的批量试错 (Rollouts)**：每一步会并行生成 512 个 rollout（即尝试 512 次）。这 512 次尝试被精准地分为 8 组，每组 64 个。
  - *与前文的联系*：这意味着，在每一步开始时，**PUCT 状态重用子程序**会从历史记录的“半成品”中，挑选出 **8个最有潜力的初始状态**。然后，针对这 8 个初始状态，模型会使用相同的上下文，分别去尝试 64 条不同的后续发展路径。
- **单步梯度更新 (On-policy)**：收集完这 512 个尝试及其对应的奖励后，系统会运用前文提到的**熵目标函数（赢者通吃）**计算梯度。由于强化学习框架底层的采样器和学习器存在机制差异，此处应用了“重要性采样比率”进行梯度校正。最终，**将这 512 个样本作为一个整体批次，执行仅仅 1 次梯度下降更新**。此过程不使用以前批次的旧数据（即严格遵循同策略 off-policy = false）。

#### 3. 护栏机制：平衡“思考”与“回答”的 Token 空间
发现类任务往往需要模型在输出复杂的解题代码（如冗长的 C++ 算法或 GPU Kernel）之前，先进行长篇大论的逻辑“思考 (Thinking)”。这极易导致上下文窗口耗尽。
- **硬性窗口限制**：`gpt-oss-120b` 在平台上的上下文窗口上限为 32,768 个 Tokens。
- **强制停止思考 (Token Forcing)**：为了防止模型“思考过度”导致没空间输出最终答案，研究人员设置了一道护栏：**将“初始提示词 + 思考过程”的总长度严格限制在 26,000 个 Tokens 以内**。一旦达到这个阈值，系统就会通过强制标记 (Token forcing)，打断模型的思考，强迫它立刻开始输出最终的解决方案代码。这确保了最后总是预留了至少 6000 多 Tokens 的充裕空间来写代码。

#### 4.1.1 Erdős 最小重叠问题 (Erdős' Minimum Overlap Problem)

这是组合数论中的一个经典问题，由 Erdős 在 1955 年提出，与序列的分布和差集有关。将 $\{1, 2, \dots, 2n\}$ 划分为基数相等为 $n$ 的两个集合 $A$ 和 $B$。将 $M_k$ 定义为满足 $a_i - b_j = k$（其中 $a_i \in A, b_j \in B$）的解的数量，并令 $M(n) = \min_{A,B} \max_k M_k$（对所有划分取最小值）。问题是限制 $c = \lim_{n\rightarrow\infty} M(n)/n$。AlphaEvolve 之前的界限为 $0.379005 < c < 0.380927$，其中上限由 Haugland 给出，下限由 [80] 给出。AlphaEvolve 和 AlphaEvolve V2 将上限改进至 $0.380924$。

#### 4.1.2 自相关不等式 (Autocorrelation Inequalities)

自相关不等式受加性组合数学的启发。改进这些不等式可以收紧常数，进而为在避免重复加性模式（加性组合数学的一个中心主题）的同时，一个集合可以有多大提供更精确的限制。与 Erdős 最小重叠问题类似，我们将构造一个阶跃函数 $f$ 来证明这些界限。

**第一自相关不等式**。对于支持在 $[-1/4, 1/4]$ 上的非负 $f$，将 $C_1$ 定义为满足以下条件的最大常数：
$$ \max_{|t| \le 1/2} (f * f)(t) \ge C_1 \left( \int f \right)^2 $$
对所有这样的 $f$ 都成立。目标是证明关于 $C_1$ 的最紧的上限；任何有效的构造 $f$ 都能证明 $C_1 \le \frac{\|f * f\|_\infty}{\|f\|_1^2}$。直到 2025 年初，已知最好的上限是 $C_1 \le 1.50973$。AlphaEvolve 将此改进至 $C_1 \le 1.5053$，AlphaEvolve V2 进一步将其改进至 $C_1 \le 1.50317$，而 ThetaEvolve 改进了 AlphaEvolve 的构造，得到了 $C_1 \le 1.50314$。

**第二自相关不等式**。对于非负 $f$，定义：
$$ C_2 = \sup_{f \ge 0} \frac{\|f * f\|_2^2}{\|f * f\|_1 \|f * f\|_\infty} $$
问题是证明关于 $C_2$ 的已知最紧下限；任何具有比率 $r$ 的有效构造 $f$ 都可以证明 $C_2 \ge r$。人类最好界限是 $C_2 \ge 0.8892$。AlphaEvolve 首先将其改进到 $C_2 \ge 0.8962$，[9] 将其改进到 $0.9015$，而 AlphaEvolve V2 进一步使用 50,000 段阶跃函数将其改进到 $C_2 \ge 0.9610$。

**结果**。我们改进了已知最好的上限，用 30000 段的阶跃函数证明了 $C_1 \le 1.50286$。比较结果见表 2。先前的 SOTA，ThetaEvolve，通过细化 AlphaEvolve V2 的构造取得了他们的结果。相比之下，TTT-Discover 从零开始发现了一个新的构造。我们在图 3 中可视化了我们的以及之前工作的阶跃函数。在第二个自相关不等式中，我们没有做出新的发现。我们的最佳结构证明的下限为 $0.959$，而 AlphaEvolve 的构造证明了更紧密的下限 $0.961$。

对于第一个不等式，早期将结果降低到 $1.510$ 左右是由于尝试和改进基于梯度的优化（例如，使用具有 softmax 参数化的 Adam）。为了将边界从大约 $1.510$ 降低到 $1.504$，策略大多使用线性规划 (LP)。后期渐进实现 SOTA 的关键洞察是：使用启发式方法仅将优化集中在接近紧（tight）的约束上——其中 LP 中的每个约束都界定卷积的一个位置。启发式方法包括挑选卷积最大的前 $K$ 个位置并仅将它们包含在 LP 中，以及从所有接近最大值的位置而不是仅从最大的单个位置计算梯度（用于基于梯度的方法）。与 AlphaEvolve 不同（其中作者暗示了使用牛顿型方法等想法），我们从未干预优化过程。

为了更好地与并发工作 ThetaEvolve 进行比较，我们还报告了使用 Qwen3-8B 的 TTT-Discover。他们使用的 Qwen3-8B 变体（DeepSeek 发布的 DeepSeek-R1-0528-Qwen3-8B）在 Tinker 上不可用。因此，我们使用了据说比 DeepSeek 变体更差的原始 Qwen 模型（Qwen/Qwen3-8B）。ThetaEvolve 报告使用了 65 个步骤，每步 512 个 rollouts（32 组，每组 16 个 rollouts），然而除此之外我们没有修改超参数，并保持 50 个步骤，每步 512 个 rollouts。对于这两个不等式，使用 Qwen3-8B 的 TTT-Discover 在使用更差模型和更小采样预算的情况下，证明了比 ThetaEvolve 更紧密的边界。

### 4.2 内核工程 (Kernel Engineering)

GPU 内核是现代 AI 的计算基础，每一个前向和后向传递最终都会作为内核代码在硬件上执行。我们将我们的方法应用于 GPU 内核优化，其中新的 SOTA 内核意味着比现有内核更快的实现。

GPUMODE 是一个用于内核开发的开放社区，也为主领域专家举办竞赛。我们在两项竞赛中测试了我们的方法：TriMul（三角形矩阵乘法），这是 AlphaFold 架构中的核心原语；以及 DeepSeek MLA（多头潜在注意力），这是 DeepSeek 推理栈中的关键组件。由于不同架构的高性能实现有所不同，TriMul 竞赛针对每种 GPU 类型（NVIDIA H100、A100、B200、AMD MI300X）分别设有排行榜。对于 MLA 竞赛，只有 MI300X 排行榜。

由于这些竞赛较早进行，我们在尊重竞赛标准的同时追溯性地评估我们的表现。我们更倾向于 GPUMODE，因为他们的排行榜通过带有人工竞赛的稳健评估框架得到了充分测试，并且他们的基准测试避免了简单操作或小输入导致开销主导运行时间的信噪比问题。

**环境**：状态 $s$ 是 GPU 内核代码。动作 $a$ 包括思考标记，然后是用 Triton 编写的内核代码。动态过程从动作中解析代码：$s' = \text{Parse}(a)$。对于初始状态，我们提供了未优化的内核（详见附录 C）。奖励与在一组固定输入形状上运行时间的几何平均值的倒数成正比（遵循排行榜），如果内核未通过正确性检查或超时，则奖励为零。我们在 Modal 上远程评估运行时间以进行扩展并确保硬件条件一致。对于 TriMul，即使我们在最终报告中评估了生成内核在 A100、B200 和 MI300X 上的运行时间，我们在训练期间也仅在 H100 上评估运行时间。由于 MI300X 在 Modal 上不可用，对于 MLA-Decode 我们使用 H200，并希望内核能够泛化到 MI300X。有关提示和环境的进一步详细信息，请参阅附录 C。

**结果**。我们在表 4 中报告了最佳内核和基线的运行时间。我们的 TriMul 内核在所有 GPU 类型的比赛中都全面达到了最先进的水平。对于 A100，我们的最佳内核比顶级人类内核快 50%，尽管我们的奖励函数没有在 A100 上为内核计时。对于所有 GPU 类型，我们始终比最佳人类提交结果实现 > 15% 的改进。最后，我们向官方 TriMul A100/H100 排行榜提交了结果。

针对 Trimul 发现的内核将频繁的逐元素操作引发的繁重内存 I/O 确定为要优化的主要瓶颈。具体而言，内核融合了：（i）输入 LayerNorm 中的操作，（ii）输入门控（input gating）中的 sigmoid 和逐元素乘法，以及（iii）输出 LayerNorm 和门控中的操作。至于计算量最大的操作，即复杂度为 $O(N^3)$ 的 matmul，内核将输入转换为 FP16，并将计算委托给 cuBLAS，以有效利用硬件的 TensorCores/MatrixCores。

**发现的 MLA-Decode 内核**。表 5 中显示的内核主要依赖于 `torch.compile()` 进行优化。具体来说，它们采用了 `torch.compile` 的特定配置。然而，这些内核没有利用 Triton 进行细粒度优化，这可能会限制进一步的改进和更灵活的使用案例。我们额外过滤并评估了明确使用 Triton 的生成内核（尽管它们的运行时间稍慢），并在附录 C 中报告。

### 4.4 单细胞分析 (Single Cell Analysis)
单细胞 RNA 测序 (RNA-seq) 旨在通过在单个细胞层面上解析生物学来帮助我们理解生物体的工作机制和生病的原因；测量每个细胞使用了哪些基因以揭示细胞类型、状态及其变化。实际上，它分离单个细胞，用唯一分子标识符 (UMI) 标记它们的 mRNA，对其进行测序，然后输出每个细胞的“基因-计数”表。RNA-seq 协议在观察到的 UMI 计数中受到测量噪声的影响。因此，去噪算法显著增加了昂贵实验的实际价值。每次测序运行的成本高达数千美元，而更好的去噪方法减少了对更深度测序的需求。

我们将我们的方法应用于最近的基准测试之一 OpenProblems，这是单细胞分析中一组重要的开放问题。我们使用了其中的去噪任务。[6] 证明，通过二项式采样将单个数据集的观察分子划分为训练集和测试集，并根据保留的测试计数评估去噪后的训练集，可以作为真实表达值准确性的代表，提供了一种无需外部基准真相数据（ground truth data）的评估框架。

**环境**。状态 $s$ 是一个算法实现。动作 $a$ 包含思考标记和随后代码。动态过程从动作中解析代码：$s' = \text{Parse}(a)$。基准测试使用两个补充指标评估去噪质量：对数归一化空间中的均方误差 (MSE)（衡量整体重建精度），以及泊松负对数似然（评估去噪计数与计数数据预期的统计特性的匹配程度）。在我们的背景下，奖励是 MSE 分数，如果它违反了我们为泊松分数添加的约束，或者算法超出了 400 秒的时间限制，则奖励为零。去噪基准提供了 3 个数据集：PBMC、Pancreas 和 Tabula Muris Senis Lung。在我们的环境中，我们使用 Pancreas 来训练我们的策略，并最终通过在保留的 PBMC 和 Tabula 数据集上运行该算法来报告性能。

**先前的最先进技术**。我们报告了由 OpenProblems 基准测试描述的 SOTA。最好的结果由 MAGIC 提供，使用了近似求解器和反向归一化。MAGIC 是一种众所周知的技术，在文献中经常使用，除了 MAGIC 之外唯一能提供良好性能的方法是 ALRA，排名第三。我们还将其与 OpenEvolve 和 Best-of-25600 进行了比较。

**免责声明**：这是一个实验性应用，展示了 TTT-Discover 发现能够在特定基准测试中表现出色的算法的能力。虽然我们发现的算法在 OpenProblems 去噪基准上优于现有方法，但基准指标本质上是不完整的，并且不能保证下游任务的生物学有效性。

**结果**。通过 TTT-Discover 获得的改进函数在两个数据集上都显示出一致的改进（见表 7）。TTT-Discover 用 MAGIC 代码初始化。TTT-Discover 添加了基因自适应转换集成（gene-adaptive transform ensembling）、低秩 SVD 细化和直接优化基准指标的对数空间抛光（polishing）步骤。

## 6. 深度思想剖析：对主流大模型 RL 范式的一次“叛逆”

这篇论文在思想上可以说是一次“返璞归真”。它明确地跳出了当前大模型推理领域最火热的 PPO / GRPO 范式，回归到了更早期的强化学习甚至经典 AI 搜索的思想路径上：

1. **跳出 PPO/GRPO 的“均分”陷阱**：当前主流的大模型 RL 都在追求“泛化”与“平均期望得分（Expected Reward）最高”。但 TTT-Discover 认为这在科学发现中是错误的：科学发现只在乎是否能**一次**打破记录（SOTA），不在乎平均失败多少次。因此它抛弃了优化期望奖励的标准做法，改用指数放大的**熵目标函数**，强行让梯度只关注那一次最高分的尝试（赢者通吃）。
2. **回归经典树搜索 (MCTS/PUCT)**：它不再强求 LLM 仅靠隐式的内部参数一次性推理出答案，而是退回到了类似 AlphaGo (2016) 时代的显式外部搜索树，用经典 PUCT 算法“站在历史的肩膀上”一步步探索。
3. **回归 REPS (2010)**：论文底层所使用的“带 KL 散度约束的熵目标函数”，本质上复活了 2010 年深度学习爆发前的经典强化学习算法 REPS (Relative Entropy Policy Search)。研究证明，在追求极限最大值时，这种经典的指数加权算法比现代的 PPO 更有效。
4. **回归局部学习 (Local Learning, 1990s)**：彻底放弃跨任务的泛化能力，针对眼前的单一测试用例进行“极端过拟合（Test-Time Training）”。这种返璞归真证明了：对于单一极其困难的突破性任务，“经典外部搜索 + 单一实例极端过拟合 + 追求极限的早期 RL 算法”的组合，能够用极小算力和廉价的开源模型，超越那些试图用巨额算力寻找通用规律的前沿闭源大模型。
